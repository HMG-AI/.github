# Support

Use the channel that matches the request:

- Reproducible bug or feature request: open the repository's issue form.
- Usage question or design discussion for a public project: use that repository's GitHub Discussions when enabled.
- Suspected vulnerability: follow `SECURITY.md`; never open a public issue.
- Private commercial or account matter: email `monkseekee@gmail.com` without including credentials.

Before filing a bug, search existing issues and include the version, operating system, minimal reproduction, expected result, actual result, and sanitized logs. Maintainers may convert support requests into tracked issues when code or documentation work is required.
